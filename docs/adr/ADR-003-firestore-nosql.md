# ADR-003: NoSQL（Firestore）の採用

- ステータス: 承認済み
- 日付: 未定

## 背景
寄付者情報、決済ステータス、チラシの場所ごとの集計など、将来的なデータの拡張性が求められる。

## 決定
Firestore（Native Mode）を採用する。

## 理由
- スキーマレスであり、将来「名刺用QR」「イベントA用QR」といった項目が増えても変更が容易
- クライアントとサーバー双方からの柔軟な参照設計が可能

## 影響
- リレーショナル設計が必要な場合は集計基盤（BigQuery等）を追加検討
- セキュリティルールとデータガバナンスを早期に整備する
