# 技術スタック

## 採用技術

### 寄付決済
- **つながる募金**（ソフトバンク運営）
  - PayPay、クレジットカード、携帯料金合算に対応
  - 手数料: 寄付額の2.4%
  - 月額固定費: なし
  - 審査期間: 約1ヶ月

### Webサイト
- **既存HP**（タダカヨ公式サイト）
  - 寄付案内ページを追加
  - つながる募金へのリンク設置

### アクセス解析
- **Google Analytics 4**
  - 流入元（QR種類）のトラッキング
  - URLパラメータ（?source=xxx）で識別

### QRコード
- 無料のQRコード生成ツールを使用
  - 例: QRコード作成サイト、Googleスプレッドシートのアドオン等

## 将来の拡張オプション

寄付規模が拡大した場合、以下への移行を検討：

| ツール | 用途 | 移行タイミング |
|--------|------|---------------|
| コングラント | 領収書自動発行、寄付者CRM | 年間寄付50万円超 |
| Googleサイト | 独自ランディングページ | 流入元別に詳細分析したい場合 |

## 採用しなかった技術

調査の結果、以下は不要と判断：

| 技術 | 当初の想定 | 不採用理由 |
|------|-----------|-----------|
| Cloud Run | APIエンドポイント | 決済API連携が不要 |
| Firestore | 寄付履歴保存 | つながる募金側で管理 |
| Secret Manager | APIキー管理 | API連携しないため不要 |
| Cloud NAT | 固定IP確保 | 決済プロバイダへの直接接続なし |
| Terraform | IaC管理 | GCPインフラを使用しない |
| Node.js / Python | バックエンド | サーバーサイド処理不要 |
